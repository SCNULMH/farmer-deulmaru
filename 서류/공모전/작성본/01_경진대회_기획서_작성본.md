# 농업·농촌 공공데이터 활용 창업경진대회 기획서 작성본

## 1. 아이디어명

**들마루 v2: 청년농 맞춤 공공데이터 기반 농업 의사결정 대시보드**

## 2. 공모 분야

- 제품·서비스 개발
- Web 서비스
- 활용 기술: 공공데이터 API, FastAPI, SQLite/PostgreSQL, AI 이미지 진단, 개인화 추천

## 3. 한 줄 소개

청년농과 귀농 초기 사용자가 지원사업, 재배 일정, 병해충 정보, AI 진단 이력을 한 화면에서 확인하고 다음 행동을 결정할 수 있도록 돕는 공공데이터 기반 농업 대시보드입니다.

## 4. 제안 배경

청년농과 귀농 초기 사용자는 농업 경험이 부족한 상태에서 지원사업 신청, 작물 재배 일정, 병해충 예찰, 교육 정보 확인을 동시에 처리해야 합니다. 현재 관련 정보는 농림축산식품 공공데이터 포털, 농사로, NCPMS, 지자체 공고 등 여러 경로에 흩어져 있어 사용자가 직접 검색하고 비교해야 하는 부담이 큽니다.

특히 초보 농업인은 다음과 같은 문제를 겪습니다.

- 내 지역과 조건에 맞는 지원사업을 찾기 어렵다.
- 작물별로 지금 해야 할 재배 작업을 놓치기 쉽다.
- 병해충 증상을 발견해도 어떤 정보와 연결해야 하는지 알기 어렵다.
- 진단 결과와 조치 이력이 개인 단위로 축적되지 않는다.

들마루 v2는 이러한 문제를 공공데이터와 사용자 맞춤형 대시보드로 해결하고자 합니다.

## 5. 아이디어 개요

들마루 v2는 사용자의 지역, 재배 작물, 관심 지원사업, 병해충 진단 이력을 기준으로 농업 의사결정에 필요한 정보를 한 화면에 정리합니다.

주요 기능은 다음과 같습니다.

- 청년농/귀농 지원사업 맞춤 추천
- 작물별 재배 일정 요약
- NCPMS 기반 병해충 예찰 정보 제공
- 이미지 기반 AI 병해충 진단
- 진단 결과 저장 및 이력 관리
- 관심 지원사업 등록 및 마감 관리

기존 들마루 프로젝트의 아이디어를 바탕으로 하되, 공모전 제출용 v2에서는 Python/FastAPI 기반으로 새로 구현하고 UI도 대시보드 중심으로 재설계합니다.

## 6. 기존 서비스와의 차별성

기존 농업 정보 서비스는 대부분 데이터 조회 중심입니다. 사용자는 지원사업, 재배 정보, 병해충 정보를 각각 다른 페이지에서 찾아야 합니다.

들마루 v2의 차별점은 다음과 같습니다.

- **개인화 흐름**: 지역, 작물, 관심사업, 진단 이력을 사용자 단위로 묶어 제공합니다.
- **의사결정 중심 UI**: 단순 목록이 아니라 “오늘 확인할 지원사업”, “이번 주 재배 작업”, “의심 병해충과 다음 조치”를 바로 보여줍니다.
- **공공데이터 연결성**: 지원사업, 작물 일정, 병해충 정보를 개별 기능이 아니라 하나의 농업 의사결정 흐름으로 연결합니다.
- **AI 진단 이력화**: 이미지 진단 결과를 일회성으로 끝내지 않고 이력으로 저장해 향후 관리와 상담에 활용할 수 있습니다.
- **시연 안정성**: 외부 API 장애 상황에서도 샘플 데이터 fallback으로 발표 시연이 가능하도록 설계합니다.

## 7. 활용 공공데이터

| 구분 | 활용 데이터 | 활용 방식 |
| --- | --- | --- |
| 지원사업 | 청년농 지원사업 API | 사용자 지역/조건 기반 지원사업 추천 및 관심 등록 |
| 재배 일정 | 농사로 작물 재배 일정 API | 사용자의 재배 작물에 맞는 주간 작업 요약 |
| 병해충 정보 | NCPMS 병해충 정보 API | AI 진단 결과와 병해충 사전/예찰 정보 연결 |
| 지역 정보 | 지자체 지원사업/지역 데이터 | 지역 기반 지원사업 필터링 및 사업화 확장 |

초기 개발 단계에서는 공모전 시연 안정성을 위해 샘플 데이터를 함께 두고, API 키는 `.env` 또는 Cloud Run 환경변수로 분리합니다.

## 8. 서비스 구성

### 8.1 사용자 흐름

1. 사용자는 로그인합니다.
2. 대시보드에서 내 지역, 재배 작물, 추천 지원사업을 확인합니다.
3. 관심 있는 지원사업을 등록합니다.
4. 작물별 이번 주 재배 작업을 확인합니다.
5. 작물 이미지를 업로드해 AI 병해충 진단을 실행합니다.
6. 진단 결과는 DB에 저장되고, 관련 병해충 정보와 다음 조치를 확인합니다.

### 8.2 화면 구성

- 로그인 화면
- 개인화 대시보드
- 지원사업 추천 카드
- 작물 재배 일정 타임라인
- AI 병해충 진단 폼
- 진단 이력 카드
- 공공데이터 출처 안내 영역

## 9. 기술 구현 계획

### 9.1 현재 구현

- Backend: FastAPI
- Frontend: Jinja2, HTML, CSS, JavaScript
- DB: SQLite 로컬 개발, Firebase Firestore 제출/배포
- 인증: 세션 기반 회원가입/로그인
- 데이터: 실제 공공데이터 API 우선 호출, 장애 시 샘플 데이터 fallback
- 배포: Firebase Hosting 진입 URL과 Cloud Run FastAPI 서비스 연결
- 추천: 사용자 지역/재배 작물 정보를 기준으로 지원사업 우선순위 산정

현재 구현 파일은 `deulmaru_v2` 폴더에 구성되어 있습니다.

### 9.2 기존 프로젝트 대비 구현 범위

기존 Spring Boot 프로젝트의 핵심 사용자 흐름은 FastAPI/Jinja2 구조로 재구성했습니다.

| 구분 | v2 반영 상태 |
| --- | --- |
| 일반 로그인/회원가입 | 세션 기반 로그인, 회원가입, Firestore 저장 구현 |
| 메인 화면 | 원본 로고, 농촌 배경, 서비스 바로가기, 추천 지원사업 영역 재구성 |
| 지원사업 | 청년농 지원사업 API, 지역/검색어/나이/기간/상태 필터, 상세보기, 관심 저장 구현 |
| 병해충 사전 | NCPMS 검색, 썸네일/예시 이미지 표시 구현 |
| 병해충 상담 QnA | NCPMS 상담사례 목록 및 상담 상세 API 연결 |
| 작물 일정 | 농사로 농작업 일정 API 연결 및 대시보드/마이페이지 표시 |
| AI 진단 | 업로드, 진단 결과 응답, 진단 이력 저장 흐름 구현 |
| 마이페이지 | 회원 정보 수정, 관심 지원사업, 진단 이력, 작물 일정 확인 구현 |
| 배포/DB | Firebase Hosting, Cloud Run, Firestore 배포 구성 완료 |

### 9.3 향후 구현 또는 단순화된 기능

- 카카오 로그인, 카카오 회원가입, 카카오 계정 연동/해제
- 실제 PyTorch `model.pth` 기반 병해충 판별 모델 완전 연결
- 원본의 성별/연령/지역별 관심 저장 통계 기반 추천 쿼리
- 교육 API, 청년사례 API 화면 연결
- 업로드 원본 이미지의 영구 저장 및 이미지 확대 이력 UI
- 원본 jQuery 기반 전체 페이지네이션/모달 UI의 세부 인터랙션
- Firebase Storage 또는 Cloudinary 기반 이미지 저장소 연결

## 10. DB 구성

초기 로컬 DB는 SQLite로 구현하고, 공모전 제출용 배포에서는 Firebase Firestore를 사용합니다. 현재 배포 환경에서는 회원, 관심 지원사업, 진단 이력이 Firestore 컬렉션 단위로 저장됩니다.

| 저장 단위 | 설명 |
| --- | --- |
| users | 사용자 계정, 지역, 재배 작물 정보 |
| interests | 관심 지원사업 등록 정보 |
| diagnosis_history | AI 병해충 진단 이력 |

Cloud Run 같은 컨테이너 배포 환경에서는 SQLite 파일 보존이 불안정할 수 있으므로, 최종 제출 환경은 Firebase Firestore를 기본 저장소로 사용합니다.

## 11. 호스팅 및 배포 계획

공모전 제출용 배포는 다음 구성으로 진행합니다.

- Hosting: Firebase Hosting
- Backend: Google Cloud Run의 FastAPI 서비스
- DB: Firebase Firestore
- Storage: Firebase Storage 또는 Cloudinary
- Secrets: Cloud Run Environment Variables

Firebase Hosting은 제출용 고정 URL을 제공하고, Cloud Run은 FastAPI 서버와 공공데이터 API 호출을 담당합니다. Firebase 콘솔에서 Firestore 데이터를 바로 확인할 수 있어 새 팀원이 함께 관리하기 쉽습니다.

## 12. 사업화 가능성

들마루 v2는 청년농, 귀농 초기 사용자, 농업 교육기관, 지자체 농업 지원센터를 주요 대상으로 확장할 수 있습니다.

사업화 방향은 다음과 같습니다.

- 청년농 대상 개인화 농업 일정/지원사업 관리 서비스
- 지자체 지원사업 홍보 및 신청 안내 플랫폼
- 농업 교육기관용 학습/실습 관리 대시보드
- 병해충 진단 이력 기반 상담 연계 서비스

장기적으로는 사용자의 작물, 지역, 진단 이력, 관심 지원사업 데이터를 기반으로 더 정밀한 추천 기능을 제공할 수 있습니다.

## 13. 기대 효과

- 초보 농업인의 정보 탐색 시간을 줄입니다.
- 지원사업 신청 기회를 놓치지 않도록 돕습니다.
- 병해충 발견 후 다음 조치를 빠르게 판단할 수 있습니다.
- 공공데이터의 활용 가치를 사용자 경험 안에서 직접 보여줍니다.
- 농업 데이터 기반 서비스의 사업화 가능성을 검증할 수 있습니다.

## 14. 개발 일정

| 기간 | 작업 |

## 15. 제출 시 강조 문장

들마루 v2는 공공데이터를 단순 조회하는 서비스가 아니라, 청년농의 실제 의사결정 흐름에 맞춰 지원사업, 재배 일정, 병해충 진단을 하나의 대시보드로 연결하는 서비스입니다.
