# 원본 프로젝트 대비 기능 정리

| 원본 기능 | v2 반영 내용 | 상태 |
| --- | --- | --- |
| 일반 로그인/회원가입 | FastAPI 세션 기반 로그인, 회원가입, Firestore 사용자 저장 | 완료 |
| 메인 화면 | 들마루 로고, 농촌 배경, 종합 서비스 바로가기, 추천 지원사업 영역 재구성 | 완료 |

| 추천 지원사업 | 사용자 지역/작물 기반 적합도 재정렬 및 추천 카드 표시 | 구현 방식 변경 |

| 병해충 상담 QnA | NCPMS 상담사례 목록과 상담 상세 API 연결 | 완료 |
| 작물 재배 일정 | 농사로 농작업 일정 API 연결, 대시보드/마이페이지 표시 | 완료 |
| AI 병해충 진단 | 이미지 업로드, 진단 결과 반환, 진단 이력 저장 | 시연 어댑터 구현 |
| 마이페이지 | 회원 정보 수정, 관심 지원사업, 진단 이력, 작물 일정 확인 | 주요 흐름 완료 |
| 기존 URL | `/supportApi/support`, `/auth/deulmaru_QnA` 등 레거시 경로 호환 | 완료 |

## 2. 원본과 다르게 구현한 부분

| 항목 | 원본 | v2 |
| --- | --- | --- |
| Backend | Spring Boot, Controller/Service/Repository 구조 | FastAPI, Router/Service/DB 추상화 구조 |
| Template | Thymeleaf + jQuery 중심 | Jinja2 + 서버 렌더링 + 일부 JavaScript |
| DB | JPA Repository 기반 관계형 DB | 로컬 SQLite, 배포 Firestore |
| 배포 | 로컬/Spring 실행 중심 | Firebase Hosting + Cloud Run |
| API 키 | `application.properties` 중심 | Cloud Run 환경변수 중심 |
| 이미지 저장 | 로컬 파일 업로드 저장 구조 | Cloud Run 제약상 원본 이미지 영구 저장 제외 |
| 추천 | 관심 저장 통계 기반 추천 쿼리 | 지역/작물/검색 조건 기반 우선순위 추천 |

## 3. 아직 구현하지 않았거나 단순화한 원본 기능

1. 카카오 로그인, 카카오 회원가입, 카카오 계정 연동/해제
2. 실제 PyTorch `model.pth`를 이용한 병해충 판별 모델 완전 연결
3. 성별/연령/지역별 관심 저장 통계를 이용한 추천 쿼리
4. 교육 API, 청년사례 API 화면 연결
5. 업로드 원본 이미지 영구 저장 및 진단 이력 이미지 확대 UI
6. 원본 마이페이지의 카카오 연동 상태 표시와 세부 탭 인터랙션 전체 복원
7. 원본 jQuery 기반 지원사업/QnA 전체 페이지네이션과 모달 세부 UI 완전 복원

## 4. 공모전 제출 설명 방향

v2는 원본 기능을 그대로 복제한 프로젝트가 아니라, 기존 들마루 아이디어를 Python/FastAPI 기반으로 재구성한 제출용 버전입니다. 핵심 시연 흐름은 회원가입, 로그인, 지역 기반 지원사업 검색, 병해충 사전 이미지 조회, QnA 상담사례, AI 진단 이력 저장, Firestore 저장, Firebase Hosting/Cloud Run 배포로 설명합니다.

미구현 기능은 제출 이후 확장 항목으로 제시합니다. 특히 카카오 로그인, 실제 AI 모델 완전 연결, 교육/청년사례 API, 이미지 저장소 연동은 사업화 가능성과 고도화 계획으로 설명하는 것이 적절합니다.
