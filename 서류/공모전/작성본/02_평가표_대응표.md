# 평가표 대응표

## 1. 공공데이터 활용

**대응 전략**

- 청년농 지원사업 API, 농사로 작물 재배 일정 API, NCPMS 병해충 API를 핵심 데이터로 사용합니다.
- 화면 하단에 공공데이터 출처를 명시합니다.
- 기능별로 어떤 공공데이터가 사용되는지 기획서와 발표자료에 표로 정리합니다.

**화면/기능 근거**

- 지원사업 추천 카드
- 작물별 재배 일정 타임라인
- NCPMS 병해충 예찰 카드
- AI 진단 결과와 병해충 정보 연결

## 2. 구체성 및 기술성

**대응 전략**

- FastAPI 기반으로 실제 실행 가능한 Web 서비스를 구현합니다.
- 로컬 SQLite와 배포용 Firebase Firestore를 모두 지원합니다.
- Firebase Hosting, Cloud Run, Firestore 기반 배포 구조를 문서화합니다.
- 기존 Spring Controller 흐름을 FastAPI 라우터와 레거시 호환 경로로 재연결합니다.

**현재 구현 근거**

- `deulmaru_v2/app/main.py`
- `deulmaru_v2/app/services/db.py`
- `deulmaru_v2/app/routers/api.py`
- `firebase.json`
- `deulmaru_v2/Dockerfile`

## 3. 독창성

**대응 전략**

- 기존 농업 정보 포털처럼 데이터를 나열하지 않고, 청년농의 오늘 할 일을 대시보드로 정리합니다.
- 지역, 작물, 관심 지원사업, 진단 이력을 한 사용자 흐름으로 연결합니다.
- AI 진단 결과를 저장하고 병해충 정보와 연결합니다.

**차별 문장**

“들마루 v2는 공공데이터 조회 서비스가 아니라 청년농의 농업 의사결정을 돕는 개인화 대시보드입니다.”

## 4. 발전 가능성 및 사업화 가능성

**대응 전략**

- Firebase Firestore, Firebase Hosting, Cloud Run, Storage 연동으로 실제 운영 가능한 구조를 제시합니다.
- 지자체/교육기관/청년농 대상 서비스로 확장 가능합니다.
- 관심 지원사업과 진단 이력 데이터가 쌓이면 추천 품질을 높일 수 있습니다.

**확장 방향**

- 지자체 지원사업 알림
- 농업 교육기관 실습 대시보드
- 병해충 상담 연계
- 작물별 작업 알림

## 5. 주제 시의성

**대응 전략**

- 청년농 정착, 귀농 초기 시행착오, 농업 정보 분산 문제를 해결합니다.
- 농식품 공공데이터 활용 활성화라는 공모전 목적과 직접 연결됩니다.
- 병해충 예찰과 지원사업 정보 접근성을 높여 농업 현장 문제 해결에 기여합니다.

## 6. 발표 시 강조 순서

1. 청년농은 정보가 부족한 것이 아니라 정보가 흩어져 있어 어렵다.
2. 들마루 v2는 공공데이터를 개인화 대시보드로 재구성한다.
3. 지원사업, 재배 일정, 병해충 진단을 한 흐름으로 연결한다.
4. SQLite/Firebase 저장 기능을 분리했고, Firebase Hosting과 Cloud Run으로 배포한다.
5. 향후 실제 API와 AI 모델을 연결하면 운영 가능한 서비스로 확장된다.
